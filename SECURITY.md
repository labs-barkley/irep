# Security Policy

IREP is currently documentation plus demonstration code; there is no production service. Still: if you find a vulnerability in the reference code, a data-leakage risk in the protocol design (e.g., a way category data could reach evaluators), or a de-anonymization vector in the audit architecture, report it privately to commons@irepprotocol.org with subject SECURITY. Acknowledgment within 7 days. Protocol-level risks are treated as seriously as code-level ones: a firewall defect in the design is a security issue here.
